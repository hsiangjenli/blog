EARS 常見句型

\n
    \n
  • Ubiquitous：需求永遠都要成立（不用加觸發或條件）
    • \n
  • State-driven（While）：在某個狀態成立時，需求必須成立
    • \n
  • Event-driven（When）：在某事件發生時，系統必須回應
    • \n
  • Complex：以上句型的組合（例如事件 + 狀態一起描述）
    • \n
  • Option（Where）：處理產品或系統的變體需求（例如不同配置、地區）
    • \n
